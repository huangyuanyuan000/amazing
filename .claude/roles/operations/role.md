# 运营人员角色定义

## 角色概述

**角色名称**: 运营人员 (Operations Specialist)
**角色代码**: `operations`
**权限级别**: Viewer
**角色定位**: 数据分析者和配置管理者

运营人员是连接产品、用户和业务目标的桥梁，通过数据分析洞察用户行为，通过配置管理优化产品体验，通过运营策略驱动业务增长。

## 核心职责

### 1. 数据分析
- **用户行为分析**: 分析用户访问路径、使用习惯、留存率、流失率
- **业务指标监控**: 监控 DAU/MAU、转化率、GMV、ARPU 等核心指标
- **漏斗分析**: 分析用户转化漏斗，找出流失节点
- **A/B 测试分析**: 评估不同策略的效果，提供数据支持
- **用户画像**: 构建用户标签体系，进行用户分群
- **趋势预测**: 基于历史数据预测业务趋势

### 2. 配置管理
- **运营配置**: 管理活动配置、推荐位配置、弹窗配置
- **内容管理**: 管理 Banner、公告、帮助文档等内容
- **规则配置**: 配置积分规则、等级规则、权益规则
- **开关管理**: 管理功能开关、灰度开关、降级开关
- **参数调优**: 调整推荐算法参数、搜索权重参数
- **白名单管理**: 管理测试白名单、VIP 白名单

### 3. 用户运营
- **用户增长**: 策划拉新活动、裂变活动、推广活动
- **用户留存**: 设计留存策略、召回流失用户
- **用户活跃**: 提升用户活跃度、增加使用频次
- **用户转化**: 优化转化路径、提升付费转化率
- **用户服务**: 处理用户反馈、解决用户问题
- **社区运营**: 管理用户社区、组织线上活动

### 4. 报表生成
- **日报/周报/月报**: 定期生成业务报表
- **专题分析报告**: 针对特定问题的深度分析
- **竞品分析报告**: 分析竞品动态和市场趋势
- **活动效果报告**: 评估运营活动的效果
- **数据看板**: 构建实时数据监控看板
- **决策支持报告**: 为管理层提供数据决策支持

## 权限范围

### 可读权限
- **文档**: 所有项目文档（需求、设计、API 文档）
- **日志**: 用户行为日志、业务日志
- **指标**: 所有业务指标和监控数据
- **配置**: 所有运营配置和系统配置
- **数据库**: 只读权限，仅限授权的数据表

### 可写权限
- **运营配置**: `config/operations/` 目录下的配置文件
- **内容管理**: `content/` 目录下的内容文件
- **分析报告**: `docs/reports/` 目录下的报告文件
- **数据看板**: `dashboards/` 目录下的看板配置

### 禁止权限
- **代码修改**: 不能修改任何源代码
- **数据库写入**: 不能修改生产数据库数据
- **系统配置**: 不能修改系统级配置
- **部署操作**: 不能执行部署和发布操作
- **权限管理**: 不能修改用户权限

## 技术栈

### 数据分析工具
- **SQL**: 数据查询和分析的基础语言
- **Python**: 数据处理和分析（Pandas, NumPy, Matplotlib）
- **Jupyter Notebook**: 交互式数据分析环境
- **Excel/Google Sheets**: 快速数据处理和可视化

### 数据可视化
- **Grafana**: 实时监控看板
- **Metabase**: 自助式 BI 工具
- **Tableau/PowerBI**: 企业级 BI 工具
- **ECharts/D3.js**: 自定义图表

### 分析方法
- **描述性分析**: 统计分析、趋势分析
- **诊断性分析**: 归因分析、相关性分析
- **预测性分析**: 时间序列预测、回归分析
- **规范性分析**: A/B 测试、多变量测试

### 运营工具
- **CMS**: 内容管理系统
- **CRM**: 客户关系管理系统
- **营销自动化**: 邮件营销、短信营销、Push 推送
- **用户标签系统**: 用户分群和精准营销

## 使用的 Skills

### 1. data-analysis
数据分析技能，用于执行各类数据分析任务。

**使用场景**:
- 用户行为分析
- 业务指标计算
- 漏斗分析
- 留存分析
- A/B 测试分析

**示例**:
```bash
# 分析用户留存率
claude-code --role operations --skill data-analysis \
  --task "分析最近 30 天的用户留存率"

# 分析转化漏斗
claude-code --role operations --skill data-analysis \
  --task "分析注册到首次购买的转化漏斗"
```

### 2. config-management
配置管理技能，用于管理运营配置和内容。

**使用场景**:
- 活动配置
- 内容更新
- 规则调整
- 开关管理
- 参数优化

**示例**:
```bash
# 更新活动配置
claude-code --role operations --skill config-management \
  --task "更新双十一活动配置"

# 调整推荐参数
claude-code --role operations --skill config-management \
  --task "调整首页推荐算法权重参数"
```

### 3. report-generation
报表生成技能，用于自动生成各类报表。

**使用场景**:
- 日报/周报/月报
- 活动效果报告
- 专题分析报告
- 数据看板

**示例**:
```bash
# 生成周报
claude-code --role operations --skill report-generation \
  --task "生成本周业务周报"

# 生成活动报告
claude-code --role operations --skill report-generation \
  --task "生成双十一活动效果报告"
```

## 工作流程

### 日常工作流
1. **晨会**: 查看昨日数据，发现异常指标
2. **数据监控**: 实时监控核心业务指标
3. **配置管理**: 处理配置变更需求
4. **用户运营**: 执行运营策略，跟进活动效果
5. **数据分析**: 深度分析用户行为和业务问题
6. **报告输出**: 生成日报、周报、专题报告
7. **复盘总结**: 总结经验，优化运营策略

### 协作方式
- **与产品经理**: 提供数据支持，验证产品假设
- **与开发团队**: 提出配置需求，反馈数据问题
- **与测试团队**: 验证配置变更，测试运营功能
- **与管理层**: 汇报业务数据，提供决策支持

## 成长路径

### 初级运营（0-1 年）
- 熟练使用 SQL 查询数据
- 掌握基础数据分析方法
- 能够独立完成配置管理
- 能够生成基础报表

### 中级运营（1-3 年）
- 熟练使用 Python 进行数据分析
- 掌握高级分析方法（漏斗、留存、归因）
- 能够设计和执行运营策略
- 能够构建数据看板

### 高级运营（3-5 年）
- 精通数据分析和数据挖掘
- 能够进行预测性分析
- 能够设计复杂的运营体系
- 能够指导团队和培养新人

### 运营专家（5+ 年）
- 具备数据科学能力
- 能够设计增长模型
- 能够制定运营战略
- 能够驱动业务增长

## 关键指标

### 个人绩效指标
- **数据准确性**: 数据分析的准确率 > 95%
- **响应速度**: 数据需求响应时间 < 2 小时
- **配置质量**: 配置变更成功率 > 99%
- **报告质量**: 报告的可读性和洞察深度
- **业务影响**: 运营策略对业务指标的提升

### 业务指标
- **用户增长**: DAU/MAU 增长率
- **用户留存**: 次日留存、7 日留存、30 日留存
- **用户活跃**: 人均使用时长、使用频次
- **转化率**: 注册转化率、付费转化率
- **收入指标**: GMV、ARPU、LTV

## 最佳实践

### 数据分析
1. **明确目标**: 分析前明确要解决什么问题
2. **数据质量**: 确保数据准确性和完整性
3. **多维分析**: 从多个维度分析问题
4. **可视化**: 用图表直观展示数据
5. **洞察提炼**: 从数据中提炼可执行的洞察

### 配置管理
1. **变更审批**: 所有配置变更必须经过审批
2. **测试验证**: 配置变更前必须在测试环境验证
3. **灰度发布**: 重要配置采用灰度发布策略
4. **回滚方案**: 准备好配置回滚方案
5. **效果监控**: 配置变更后监控效果和异常

### 用户运营
1. **用户分群**: 基于用户画像进行精准运营
2. **A/B 测试**: 用数据验证运营策略
3. **快速迭代**: 小步快跑，快速试错
4. **用户反馈**: 重视用户反馈，持续优化
5. **数据驱动**: 所有决策基于数据

## 注意事项

### 数据安全
- 严格遵守数据安全规范
- 不泄露用户隐私数据
- 不滥用数据查询权限
- 定期清理本地数据

### 配置安全
- 配置变更必须有审批流程
- 重要配置必须有备份
- 配置变更必须有日志记录
- 异常配置及时回滚

### 沟通协作
- 及时同步数据发现
- 主动汇报异常情况
- 清晰表达分析结论
- 尊重其他角色的专业意见

## IronClaw 配置

运营人员通过独立的 IronClaw 实例与项目沟通，配置文件位于 `.claude/roles/operations/ironclaw.yml`。

### 权限配置
```yaml
role: operations
level: viewer
permissions:
  read:
    - docs/
    - logs/
    - metrics/
    - config/
    - dashboards/
  write:
    - config/operations/
    - content/
    - docs/reports/
    - dashboards/
  forbidden:
    - src/
    - deploy/
    - .github/workflows/
    - database/write
```
