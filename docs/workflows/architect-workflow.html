👑 架构师工作流程

← 返回主页 | 📦 开发流程
1 需求接收与理解
👑 架构师

架构师接收产品经理提交的需求，进行初步分析。

$ claude code --role architect
> 查看新需求：用户管理模块
💡 关键点：
  • 理解业务目标和用户价值
  • 识别技术约束和依赖
  • 评估实现复杂度
  • 明确验收标准
2 AI 方案生成
AI Agent

AI 基于需求自动生成多种技术方案供架构师选择。

方案 A：微服务架构

独立服务，易扩展，适合大规模

方案 B：单体架构

简单快速，适合快速迭代

方案 C：混合架构

核心单体+边缘微服务

3 架构决策
👑 架构师

架构师基于经验和项目实际情况，选择最优方案。

🎯 决策维度：
  • 性能要求：QPS、延迟、并发量
  • 可扩展性：未来增长预期
  • 团队能力：技术栈熟悉度
  • 成本预算：开发和运维成本
  • 时间约束：上线时间要求
$ claude code
> 我选择方案 B：单体架构
> 理由：团队规模小，需要快速迭代，后期可拆分
4 技术栈选型
👑 架构师
AI Agent

AI 推荐技术栈，架构师确认或调整。

✅ AI 推荐技术栈：
  • 前端：React + TypeScript + Ant Design
  • 后端：FastAPI + Python 3.11
  • 数据库：PostgreSQL 15
  • 缓存：Redis 7
  • 消息队列：RabbitMQ
$ claude code
> 确认技术栈
> 调整：将 RabbitMQ 改为 Kafka（团队更熟悉）
5 架构设计
👑 架构师

设计系统架构，包括模块划分、接口定义、数据流等。

💡 设计内容：
  • 系统架构图
  • 模块划分和职责
  • API 接口设计
  • 数据库表结构
  • 部署架构
6 AI 辅助设计
AI Agent

AI 根据架构师的设计意图，自动生成详细设计文档。

✅ AI 自动生成：
  • API 接口文档（OpenAPI 规范）
  • 数据库 DDL 脚本
  • 架构图（Mermaid 格式）
  • 部署配置文件
7 设计评审
👑 架构师
技术负责人

组织技术评审会议，讨论架构设计。

🎯 评审要点：
  • 架构合理性
  • 技术可行性
  • 性能预估
  • 风险识别
  • 成本评估
8 任务分解
👑 架构师
AI Agent

将架构设计分解为具体开发任务。

$ claude code
> 将用户管理模块分解为开发任务
✅ 任务列表：
  • 前端：用户列表页面
  • 前端：用户详情页面
  • 后端：用户 CRUD API
  • 后端：权限验证中间件
  • 数据库：用户表创建
  • 测试：单元测试
  • 测试：集成测试
9 开发指导
👑 架构师

在开发过程中提供技术指导和问题解答。

💡 指导内容：
  • 解答技术疑问
  • Code Review
  • 性能优化建议
  • 架构调整决策
10 架构验证
👑 架构师

验证实现是否符合架构设计。

$ claude code
> 检查代码是否符合架构设计
> 运行架构一致性测试
✅ 验证项：
  • 模块依赖关系
  • 接口规范遵守
  • 性能指标达标
  • 安全规范落实
