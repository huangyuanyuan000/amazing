🔄 开发工作流程

1
需求接入
产品经理通过 Claude Code CLI 提交需求 → Common Agent 接收并分析需求类型
2
Agent 路由
Common Agent 根据需求类型路由到对应的业务 Agent (Compute/Data/Training/Model-Service)
3
Sub-Agent 编排
业务 Agent 的 Orchestrator 分解任务，调度相应的 Sub-Agents 执行
4
代码生成
Sub-Agents 调用 Skills 生成代码 → 前端/后端/Go 服务代码自动生成
5
代码审核
Review Agent 自动审核代码（安全/性能/风格） → 不通过则返回修改
6
测试验证
测试工程师通过 CLI 触发自动化测试 → 单测/集成测试/E2E 测试
7
部署上线
运维工程师通过 CLI 一键部署 → Docker Compose (本地) / K8s (生产)
8
监控运营
运营人员通过 CLI 查看数据 → 监控告警 → 问题反馈 → 进入 Bug 修复流程

🐛 Bug 修复与回滚流程

1
问题发现
监控告警 / 用户反馈 → 运维/测试通过 CLI 提交 Bug 报告
2
问题定位
Review Agent 分析日志和代码 → 定位问题代码位置和版本
3
回滚决策
严重问题：立即回滚到上一稳定版本 → 非严重：进入修复流程
4
代码修复
对应 Agent 生成修复代码 → Review Agent 审核 → 测试验证
5
灰度发布
小流量验证 → 逐步扩大 → 全量发布 → 监控观察

🚀 部署模式

📦 本地开发 (Docker Compose)

适用场景：本地开发、功能测试

$ make deploy-local

特点：

  • 一键启动所有服务
  • 自动初始化数据库
  • 支持热重载
  • 本地端口映射
☸️ 生产环境 (Kubernetes)

适用场景：生产部署、高可用

$ make deploy-k8s

特点：

  • 自动扩缩容
  • 滚动更新
  • 健康检查
  • 服务发现
🔒 私有化部署 (离线)

适用场景：内网环境、私有云

$ make deploy-offline

特点：

  • 离线镜像包
  • 自动环境检测
  • 数据库自动适配
  • 一键初始化
🐳 Docker 快速部署

适用场景：快速验证、演示

$ make deploy-docker

特点：

  • 单机部署
  • 资源占用小
  • 快速启动
  • 易于清理

⚡ 快速开始

1. 克隆项目

$ git clone https://github.com/your-org/amazing.git
$ cd amazing

2. 选择部署模式

# 本地开发
$ make deploy-local

# 生产环境
$ make deploy-k8s

# 私有化部署
$ make deploy-offline

3. 访问服务

  • 前端： http://localhost:3000
  • 后端 API： http://localhost:8000
  • API 文档： http://localhost:8000/docs
  • 监控面板： http://localhost:9090

4. 使用 CLI 开发

# 使用 Claude Code CLI (推荐)
$ claude code

# 使用 Codex CLI (降级)
$ codex cli

# 使用 Codex Desktop
$ codex desktop
