# Test Design Skill - 测试用例设计方法论

## 功能描述
提供测试用例设计方法论、覆盖率策略和测试分层方案。

## 触发方式
- 新功能测试设计
- 测试覆盖率提升
- 测试策略制定

## 核心内容

### 1. 测试金字塔
```
        /  E2E  \        少量，验证关键流程
       / 集成测试 \       中量，验证模块交互
      /  单元测试  \      大量，验证函数逻辑
```

### 2. 测试设计方法
| 方法 | 适用场景 | 说明 |
|------|----------|------|
| 等价类划分 | 输入验证 | 有效/无效等价类 |
| 边界值分析 | 数值范围 | 边界值 ± 1 |
| 决策表 | 多条件组合 | 条件-动作矩阵 |
| 状态转换 | 状态机 | 状态-事件-转换 |

### 3. 覆盖率目标
| 层级 | 覆盖率 | 说明 |
|------|--------|------|
| 单元测试 | > 80% | 核心逻辑 100% |
| 集成测试 | > 60% | API 端点全覆盖 |
| E2E 测试 | 关键路径 | 核心业务流程 |

### 4. 测试命名规范
```
test_{被测方法}_{场景}_{预期结果}
例: test_create_user_with_valid_data_returns_201
```

## 示例
| 用例 | 输入 | 预期 |
|------|------|------|
| 正常注册 | 有效用户名+密码+邮箱 | 201 Created |
| 重复用户名 | 已存在的用户名 | 409 Conflict |
| 无效邮箱 | 格式错误的邮箱 | 400 Bad Request |

## 进化能力
- 测试模式库持续扩充
- 从 Bug 中反推缺失用例
- 测试效率持续优化
