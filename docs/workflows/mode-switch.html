⚠️ 权限要求

只有架构师 (Architect) 有权限切换 AI 模式

  • ✅ 架构师：拥有完整的模式切换权限
  • ❌ 产品经理、开发、测试、运维：无权切换模式
  • 📝 所有切换操作都会被记录和审计
  • 🔗 模式切换需要关联具体需求ID

模式对比

🤖 全自动模式 (Full-Auto)

AI 角色：执行者

人工角色：监督者

决策权：AI 自主决策

执行权：AI 自动执行

适用场景：

  • 成熟流程
  • 低风险任务
  • Bug 修复
  • 代码重构
👨‍💻 半自动模式 (Semi-Auto)

AI 角色：辅助者

人工角色：决策者

决策权：人工决策

执行权：人工批准后执行

适用场景：

  • 新项目启动
  • 高风险任务
  • 架构设计
  • 技术选型

完整切换流程

1 切换到架构师角色
👑 架构师

首先需要切换到架构师角色，只有架构师才有权限切换模式。

$ python3 scripts/amazing-cli.py role set architect
✅ 角色切换成功
当前角色: 架构师 (Architect)
2 评估需求
👑 架构师

架构师评估需求的复杂度、风险等级，决定使用哪种模式。

💡 评估维度：
  • 技术复杂度：高/中/低
  • 业务风险：高/中/低
  • 团队熟悉度：熟悉/一般/不熟悉
  • 时间要求：紧急/正常/宽松
3 执行模式切换
👑 架构师

使用 CLI 命令切换模式，需要提供需求ID和切换原因。

切换到全自动模式：

$ python3 scripts/mode_cli.py mode set full-auto \
-r REQ-001 \
--reason "成熟功能，团队熟悉，风险可控"

切换到半自动模式：

$ python3 scripts/mode_cli.py mode set semi-auto \
-r REQ-002 \
--reason "新项目，需要人工把关"
4 系统验证权限
系统

系统自动验证用户是否有权限切换模式。

检查用户角色
读取 .claude/user.json 文件
↓
验证权限
role === "architect" ?
↓
✅ 是架构师
继续执行
❌ 不是架构师
拒绝操作
❌ 权限不足示例：
权限不足：只有架构师可以切换模式（当前角色: frontend）

💡 提示: 请先切换到架构师角色
python3 scripts/amazing-cli.py role set architect
5 记录切换信息
系统

系统记录模式切换的详细信息，用于审计和追溯。

记录内容：

{ "timestamp": "2025-03-15T10:30:00", "operator": "architect", "from": "semi-auto", "to": "full-auto", "requirement_id": "REQ-001", "reason": "成熟功能，团队熟悉，风险可控" }

记录位置：

  • .agents/config.json - 配置文件中的 modeHistory
  • .agents/logs/mode_changes.log - 审计日志文件
6 AI 按新模式执行
AI Agent

AI 根据新的模式设置执行任务。

全自动模式执行
  • AI 自主决策技术方案
  • AI 自动生成代码
  • AI 自动运行测试
  • 关键节点通知架构师
  • 异常时暂停等待人工
半自动模式执行
  • AI 生成多种方案
  • 架构师选择方案
  • AI 生成代码草稿
  • 架构师审查代码
  • 架构师批准后执行

查看切换历史

查看模式切换历史
$ python3 scripts/mode_cli.py mode history

输出示例：

时间 切换 需求ID 原因 操作者
2025-03-15 10:30 semi-auto → full-auto REQ-001 成熟功能，风险可控 architect
2025-03-14 15:20 full-auto → semi-auto REQ-002 新项目，需要把关 architect

按需求维度切换

不同需求使用不同模式

每个需求可以独立设置模式，互不影响。

# 需求 REQ-001: 用户登录功能（成熟功能，低风险）
$ python3 scripts/mode_cli.py mode set full-auto -r REQ-001

# 需求 REQ-002: 支付系统（高风险，需要把关）
$ python3 scripts/mode_cli.py mode set semi-auto -r REQ-002

# 需求 REQ-003: 数据报表（低风险，快速开发）
$ python3 scripts/mode_cli.py mode set full-auto -r REQ-003
💡 优势：
  • 灵活性高：不同需求可以使用不同策略
  • 风险可控：高风险需求使用半自动模式
  • 效率优化：低风险需求使用全自动模式
  • 可追溯：每个需求的模式切换都有记录
