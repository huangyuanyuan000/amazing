# 测试工程师 (QA) 指南

## 角色定位

测试工程师负责软件质量保证，包括测试用例设计、自动化测试、Bug 追踪和质量报告。

## 核心职责

### 1. 测试用例设计
- 功能测试用例
- 边界条件测试
- 异常场景测试
- 性能测试用例

### 2. 自动化测试
- 单元测试编写
- 集成测试编写
- E2E 测试编写
- 测试脚本维护

### 3. Bug 追踪
- Bug 复现
- Bug 报告
- Bug 验证
- 回归测试

### 4. 质量报告
- 测试覆盖率统计
- Bug 统计分析
- 质量评估报告
- 测试总结

## 工作流程

### 测试流程

```
1. 接收测试任务
   ↓
2. 设计测试用例
   ├── 功能测试
   ├── 性能测试
   └── 安全测试
   ↓
3. 执行测试
   ├── 手动测试
   └── 自动化测试
   ↓
4. Bug 报告
   ├── 记录 Bug
   ├── 分配优先级
   └── 提交开发
   ↓
5. 回归测试
   ├── 验证修复
   └── 确认关闭
```

## 使用工具

### 测试框架
- **Pytest**: Python 单元测试
- **Jest**: JavaScript 单元测试
- **Playwright**: E2E 测试
- **JMeter**: 性能测试

### CLI 命令

```bash
# 切换到 QA 角色
python3 scripts/amazing-cli.py role set qa

# 查看待测试任务
python3 scripts/amazing-cli.py tasks --role qa

# 运行测试
pytest
npm run test
npm run test:e2e
```

## 权限说明

- ✅ 测试用例设计
- ✅ 测试执行
- ✅ Bug 报告
- ✅ 质量审查
- ❌ 代码开发
- ❌ 部署操作

## 最佳实践

1. **测试用例要全面**：覆盖正常流程、异常流程、边界条件
2. **自动化优先**：重复性测试尽量自动化
3. **及时报告 Bug**：发现问题立即记录
4. **回归测试必做**：每次修复后都要验证
