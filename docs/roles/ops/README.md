# 运维工程师 (Ops) 指南

## 角色定位

运维工程师负责系统部署、监控、故障排查和性能优化。

## 核心职责

### 1. 部署管理
- 本地环境部署
- Docker 容器化部署
- Kubernetes 集群部署
- CI/CD 流水线配置

### 2. 监控告警
- 服务监控配置
- 性能指标监控
- 日志收集分析
- 告警规则配置

### 3. 故障排查
- 服务故障诊断
- 性能问题定位
- 日志分析
- 问题修复

### 4. 性能优化
- 资源优化
- 性能调优
- 容量规划
- 成本优化

## 工作流程

### 部署流程

```
1. 接收部署任务
   ↓
2. 环境准备
   ├── 检查资源
   ├── 配置环境
   └── 准备镜像
   ↓
3. 执行部署
   ├── 本地部署
   ├── Docker 部署
   └── K8s 部署
   ↓
4. 验证部署
   ├── 健康检查
   ├── 冒烟测试
   └── 监控配置
   ↓
5. 上线监控
   ├── 观察指标
   ├── 查看日志
   └── 处理告警
```

## 使用工具

### 部署工具
- **Docker**: 容器化
- **Kubernetes**: 容器编排
- **Helm**: K8s 包管理
- **Ansible**: 自动化运维

### 监控工具
- **Prometheus**: 指标监控
- **Grafana**: 可视化
- **ELK Stack**: 日志分析
- **AlertManager**: 告警管理

### CLI 命令

```bash
# 切换到 Ops 角色
python3 scripts/amazing-cli.py role set ops

# 本地部署
make deploy-local

# Docker 部署
make deploy-docker

# K8s 部署
make deploy-k8s

# 查看服务状态
kubectl get pods -n amazing
docker-compose ps
```

## 权限说明

- ✅ 部署执行
- ✅ 监控配置
- ✅ 故障排查
- ✅ 基础设施管理
- ❌ 代码开发
- ❌ 需求变更

## 最佳实践

1. **部署前检查**：确认资源充足、配置正确
2. **灰度发布**：先小流量验证，再全量发布
3. **监控先行**：部署前配置好监控告警
4. **快速回滚**：出现问题立即回滚
5. **文档记录**：记录部署步骤和问题
