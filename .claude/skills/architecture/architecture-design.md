# Architecture Design Skill - 架构设计方法论

## 功能描述
提供系统架构设计的方法论、模式选择指南和 ADR 模板。

## 触发方式
- 新项目架构设计
- 架构评审
- 架构演进规划

## 核心内容

### 1. 架构设计流程
1. **需求分析**: 功能需求 + 非功能需求（性能、安全、可用性）
2. **领域建模**: 识别核心域、支撑域、通用域
3. **架构选型**: 根据场景选择架构模式
4. **模块划分**: 高内聚、低耦合原则
5. **接口设计**: API 契约优先
6. **数据设计**: 数据流向、存储策略
7. **部署设计**: 部署拓扑、网络架构

### 2. 架构模式选择
| 模式 | 适用场景 | 优势 | 劣势 |
|------|----------|------|------|
| 分层架构 | 中小型项目 | 简单清晰 | 层间耦合 |
| 微服务 | 大型分布式 | 独立部署 | 运维复杂 |
| 事件驱动 | 异步处理 | 解耦 | 调试困难 |
| CQRS | 读写分离 | 性能优化 | 复杂度高 |
| 六边形 | 领域驱动 | 可测试性 | 学习曲线 |

### 3. ADR 模板
```markdown
# ADR-{编号}: {标题}
## 状态: {Proposed | Accepted | Deprecated}
## 背景: {问题描述}
## 决策: {具体决策}
## 理由: {选择原因}
## 后果: {影响分析}
```

## 示例
### 电商平台架构
```
用户层 → API Gateway → 微服务集群
                        ├── 用户服务
                        ├── 商品服务
                        ├── 订单服务
                        └── 支付服务
                        ↓
                   消息队列 → 异步处理
                        ↓
                   数据层（PostgreSQL + Redis + ES）
```
