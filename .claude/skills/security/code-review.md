# Code Review Skill - 代码审查清单

## 功能描述
提供代码审查清单、常见反模式检测和审查最佳实践。

## 触发方式
- Pull Request 审查
- 代码质量检查
- 重构评估

## 核心内容

### 1. 审查维度
| 维度 | 权重 | 检查项 |
|------|------|--------|
| 正确性 | 30% | 逻辑正确、边界处理 |
| 安全性 | 25% | 注入、越权、泄露 |
| 可维护性 | 20% | 命名、结构、复杂度 |
| 性能 | 15% | N+1、内存泄漏 |
| 测试 | 10% | 覆盖率、用例质量 |

### 2. 常见反模式
| 反模式 | 问题 | 改进 |
|--------|------|------|
| God Class | 类职责过多 | 拆分为多个类 |
| N+1 查询 | 循环查询数据库 | 批量查询/JOIN |
| 硬编码 | 配置写死在代码中 | 配置文件/环境变量 |
| 过深嵌套 | if/for > 3 层 | 提前返回/提取方法 |
| 全局状态 | 可变全局变量 | 依赖注入 |

### 3. 审查流程
1. 理解变更目的 → 2. 检查架构一致性 → 3. 逐文件审查 → 4. 运行测试 → 5. 输出意见

### 4. 审查意见分级
| 级别 | 含义 |
|------|------|
| BLOCK | 必须修复才能合并 |
| ERROR | 需要修复 |
| WARNING | 建议修复 |
| INFO | 仅供参考 |

## 示例
```
[ERROR] 缺少输入验证 - src/api/users.py:30
问题: 用户输入未经验证直接使用
建议: 添加 Pydantic 模型验证
```

## 进化能力
- 审查规则库持续扩充
- 从历史审查中学习模式
- 误报率持续降低
