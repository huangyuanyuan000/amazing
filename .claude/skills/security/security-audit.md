# Security Audit Skill - 安全审计清单

## 功能描述
提供安全审计清单、OWASP Top 10 检查项和安全编码规范。

## 触发方式
- 安全审查
- 上线前安全检查
- 代码审查（安全维度）

## 核心内容

### 1. OWASP Top 10 检查
| 编号 | 风险 | 检查项 |
|------|------|--------|
| A01 | 访问控制失效 | 权限检查、越权测试 |
| A02 | 加密失败 | 敏感数据加密、TLS |
| A03 | 注入 | SQL/NoSQL/命令注入 |
| A04 | 不安全设计 | 威胁建模、安全设计 |
| A05 | 安全配置错误 | 默认配置、错误信息 |
| A06 | 过时组件 | 依赖漏洞扫描 |
| A07 | 认证失败 | 弱密码、暴力破解 |
| A08 | 数据完整性 | 反序列化、CI/CD |
| A09 | 日志不足 | 审计日志、监控 |
| A10 | SSRF | 服务端请求伪造 |

### 2. 安全编码规范
- 输入验证: 所有外部输入必须验证
- 参数化查询: 禁止 SQL 拼接
- 输出编码: 防止 XSS
- 密码哈希: bcrypt/argon2
- 敏感数据: 加密存储，脱敏展示

### 3. 安全审计清单
- [ ] 认证机制安全（JWT/Session）
- [ ] 授权检查完整（RBAC/ABAC）
- [ ] 输入验证（类型、长度、格式）
- [ ] SQL 注入防护
- [ ] XSS 防护
- [ ] CSRF 防护
- [ ] 敏感数据加密
- [ ] 日志审计完整
- [ ] 依赖漏洞扫描
- [ ] 错误信息不泄露内部细节

## 示例
```
[CRITICAL] SQL 注入 - src/api/users.py:45
问题: 用户输入直接拼接 SQL
修复: 使用参数化查询
```
