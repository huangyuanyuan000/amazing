# 运营人员 (Operation) 指南

## 角色定位

运营人员负责数据分析、用户运营和配置管理。

## 核心职责

### 1. 数据分析
- 用户行为分析
- 业务数据统计
- 报表生成
- 数据可视化

### 2. 用户运营
- 用户增长
- 用户留存
- 用户反馈处理
- 用户满意度调查

### 3. 配置管理
- 功能开关配置
- 参数配置
- 规则配置
- 内容管理

### 4. 运营活动
- 活动策划
- 活动执行
- 效果评估
- 优化迭代

## 工作流程

### 数据分析流程

```
1. 确定分析目标
   ↓
2. 数据收集
   ├── 用户数据
   ├── 行为数据
   └── 业务数据
   ↓
3. 数据分析
   ├── 统计分析
   ├── 趋势分析
   └── 对比分析
   ↓
4. 生成报告
   ├── 数据可视化
   ├── 结论总结
   └── 建议输出
   ↓
5. 优化迭代
   ├── 制定策略
   └── 执行优化
```

## 使用工具

### 数据分析工具
- **SQL**: 数据查询
- **Python**: 数据处理
- **Jupyter Notebook**: 数据分析
- **Excel**: 报表制作

### CLI 命令

```bash
# 切换到运营角色
python3 scripts/amazing-cli.py role set operation

# 查看用户统计
python3 scripts/amazing-cli.py stats users

# 导出数据
python3 scripts/amazing-cli.py export users --format csv

# 查看配置
python3 scripts/amazing-cli.py config list

# 更新配置
python3 scripts/amazing-cli.py config set feature.new_ui true
```

## 权限说明

- ✅ 数据分析
- ✅ 配置管理
- ✅ 用户运营
- ❌ 代码开发
- ❌ 部署操作
- ❌ 数据库修改

## 最佳实践

1. **数据驱动决策**：基于数据做决策，而非主观判断
2. **定期分析**：建立定期数据分析机制
3. **关注核心指标**：聚焦关键业务指标
4. **快速响应**：及时处理用户反馈
5. **持续优化**：根据数据不断优化运营策略
