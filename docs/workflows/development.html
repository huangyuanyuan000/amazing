🔄 开发工作流程

← 返回主页 | 📦 代码仓库
1 需求提交
产品经理

产品经理通过 Claude Code CLI 提交需求文档。

$ claude code
> 我要提交一个新需求：添加用户管理模块
💡 提示：需求描述要清晰，包含功能点、验收标准、优先级等信息。
2 需求分析与路由
Common Agent

Common Agent 自动分析需求类型，路由到对应的业务 Agent。

  • 通用功能 → Common Agent
  • 算力相关 → Compute Agent
  • 数据相关 → Data Agent
  • 训练相关 → Training Agent
  • 模型服务 → Model Service Agent
3 任务分解与编排
Orchestrator Sub-Agent

业务 Agent 的 Orchestrator 将需求分解为具体任务，分配给相应的 Sub-Agents。

✅ 自动生成：
  • 前端组件代码
  • 后端 API 接口
  • 数据库表结构
  • 单元测试用例
4 代码生成
前端开发
后端开发

Sub-Agents 调用对应的 Skills 生成代码。

# 前端代码生成
$ claude code
> 生成用户管理页面

# 后端代码生成
$ claude code
> 生成用户管理 API
5 代码审核
Review Agent

Review Agent 自动审核生成的代码。

  • Security Agent：安全漏洞检测
  • Performance Agent：性能问题检测
  • Style Agent：代码风格检查
  • Code Review Agent：代码质量审核
💡 提示：审核不通过会自动返回修改建议，Sub-Agent 会自动修复。
6 测试验证
测试工程师

测试工程师通过 CLI 触发自动化测试。

$ make test

# 或者
$ claude code
> 运行用户管理模块的测试用例
✅ 测试类型：
  • 单元测试 (Unit Test)
  • 集成测试 (Integration Test)
  • E2E 测试 (End-to-End Test)
7 部署上线
运维工程师

运维工程师通过 CLI 一键部署。

# 本地环境
$ make deploy-local

# 生产环境
$ make deploy-k8s
8 监控与反馈
运营人员
运维工程师

运营和运维人员监控系统运行状态。

  • 实时监控服务状态
  • 查看业务数据
  • 接收告警通知
  • 问题反馈
