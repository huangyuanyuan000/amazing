# Query Optimization Skill - 查询优化方法论

## 功能描述
提供 SQL 查询优化方法论、EXPLAIN 分析指南和性能基准。

## 触发方式
- 慢查询优化
- 性能审查
- 数据库调优

## 核心内容

### 1. 优化流程
1. 识别慢查询（> 100ms）→ 2. EXPLAIN 分析 → 3. 识别瓶颈 → 4. 优化方案 → 5. 验证效果

### 2. 常见问题与解决
| 问题 | 原因 | 解决方案 |
|------|------|----------|
| N+1 查询 | 循环单条查询 | JOIN 或批量查询 |
| 全表扫描 | 缺少索引 | 添加合适索引 |
| 大偏移分页 | OFFSET 过大 | 游标分页 |
| SELECT * | 查询所有字段 | 只查需要的字段 |
| 未命中索引 | 函数/类型转换 | 调整查询条件 |

### 3. EXPLAIN 关键指标
- `Seq Scan`: 全表扫描（需关注）
- `Index Scan`: 索引扫描（正常）
- `Nested Loop`: 嵌套循环（大数据量需优化）
- `actual time`: 实际执行时间

### 4. 性能基准
| 查询类型 | 目标 | 告警阈值 |
|----------|------|----------|
| 单条查询 | < 10ms | > 100ms |
| 列表查询 | < 50ms | > 500ms |
| 聚合查询 | < 200ms | > 2000ms |

## 示例
```sql
-- 优化前（大偏移）
SELECT * FROM orders ORDER BY id LIMIT 20 OFFSET 10000;
-- 优化后（游标分页）
SELECT * FROM orders WHERE id > :last_id ORDER BY id LIMIT 20;
```
