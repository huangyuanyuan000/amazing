# Tech Selection Skill - 技术选型评估框架

## 功能描述
提供技术选型的评估框架、对比矩阵模板和决策流程。

## 触发方式
- 新项目技术栈选择
- 技术栈升级/迁移
- 新组件/中间件引入

## 核心内容

### 1. 评估维度
| 维度 | 权重 | 评估要点 |
|------|------|----------|
| 功能匹配 | 25% | 是否满足核心需求 |
| 性能 | 20% | 吞吐量、延迟、资源占用 |
| 生态 | 15% | 社区、文档、第三方库 |
| 团队适配 | 15% | 学习曲线、团队经验 |
| 成本 | 10% | 许可、运维、迁移成本 |
| 安全 | 10% | 漏洞历史、安全特性 |
| 可扩展性 | 5% | 水平扩展、插件机制 |

### 2. 常见技术栈推荐
| 场景 | 后端 | 前端 | 数据库 | 缓存 |
|------|------|------|--------|------|
| 快速原型 | Python/FastAPI | React/Next.js | PostgreSQL | Redis |
| 高性能 | Go/Gin | React | PostgreSQL | Redis |
| 企业级 | Java/Spring | React/Angular | MySQL | Redis |
| AI 平台 | Python/FastAPI | React | PG + MongoDB | Redis |

### 3. 选型报告模板
```markdown
## 技术选型报告: {组件名}
### 需求背景
### 候选方案对比
| 方案 | 功能 | 性能 | 生态 | 团队 | 总分 |
### 推荐方案及理由
### 风险与缓解
### POC 验证计划
```

## 示例
### 后端框架选型
| 方案 | FastAPI | Django | Express | Gin |
|------|---------|--------|---------|-----|

## 进化能力
- 技术评估数据持续更新
- 选型效果反馈追踪
